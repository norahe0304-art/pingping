# Claude Code协工作17条最佳实践

## 内容摘要

1. **核心理念转变**：投资 setup（环境配置）比投资 prompting 更有回报，ChatGPT 奖励 prompt engineering，Cowork 奖励 system engineering
2. **三层文档结构**：用 _MANIFEST.md 建立 Canonical/Domain/Archival 三层结构管理知识
3. **Global Instructions**：作为永久操作系统，提供持续可用的基础指令
4. **三个持久上下文文件**：about-me.md（个人资料）、brand-voice.md（品牌调性）、working-style.md（工作风格）
5. **Folder Instructions**：项目级上下文配置，针对不同项目设置不同背景
6. **有意识地控制读取范围**：不要让 AI 读取全部内容，明确范围提高效率
7. **定义 end state**：任务设计时定义最终状态，而不是具体步骤
8. **执行前先 show plan**：让 AI 先展示计划再执行，便于人类审核
9. **处理不确定性**：明确告诉 AI 不确定时应该怎么做
10. **批量处理**：相关任务放在一个 session 里处理，减少上下文切换
11. **用 subagent 并行**：用 subagent 并行处理多个任务
12. **定时任务**：用 /schedule 设置定时执行的任务
13. **外置可复用逻辑**：一次构建，多次运行，把逻辑外置到文件
14. **真自动化**：/schedule + connectors 组合实现真正的自动化
15. **组合插件**：组合多个插件实现复杂功能
16. **Custom skills**：为特定工作流建立自定义技能
17. **Plugin Management**：用对话式方式构建插件

## 我们可以做什么

1. **建立个人 Manifest**：参考三层结构建立自己的知识管理体系
2. **完善三个持久文件**：创建或完善 about-me、brand-voice、working-style 三个文件
3. **设置定时任务**：把重复性工作用定时任务自动化
4. **定义任务 end state**：以后派任务时先想清楚最终状态
5. **测试 subagent 并行**：尝试用 subagent 并行处理多个相关任务
