## 阅读理解

这条推文分享了一个用 OpenClaw 完成大型项目的实用技巧。作者 @CryptoPainter 的核心建议是：不要让 Agent 直接工作，而是让它自己部署一个子代理，然后让它监督子代理工作。这样不论子代理出了什么问题和 bug，总控 Agent 都能唤醒它并让它继续工作。到后期还可以再加一个测试代理，相当于你同时有了一个产品经理、一个程序员以及一个测试员。作者提到可以看到这三个 Agent 在你睡觉时都在互相沟通协作，总代理已经对部署流程非常熟练。

## 实战洞察

这个"总控 Agent + 子代理"的分层架构模式解决了 AI 编程中的一个核心问题：单个 Agent 在长时间复杂任务中容易迷失方向或卡住。通过引入监督层，总控 Agent 可以在子代理出错时介入纠正，大幅提高了任务完成率。三角色分工（产品经理、程序员、测试员）的设计也符合软件工程的基本协作模式。这个方法特别适合需要持续运行数小时的大型项目，让 Agent 系统具备了自我修复和质量保障的能力。
