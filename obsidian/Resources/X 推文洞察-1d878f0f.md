## 阅读理解

这条推文探讨了多 Agent 系统中安全规则管理的核心痛点。作者 @ohxiyu 提出了一个"SECURITY.md 下沉设计"的理念，核心观点是：当你有多个 AI Agent 各自拥有人格定义、分工和权限时，安全规则往往散落在各个配置文件里，导致有的 Agent 配了"不要删文件"，有的完全没配，有的知道不能碰密钥目录，有的压根没这个概念。更要命的是，改一条安全规则就得挨个去改每个 Agent 的配置。作者主张的安全原则是"多做 1%，安全性就可能从 0% 提高到 99%"，通过将安全规则集中到一个 SECURITY.md 文件中，让所有 Agent 统一继承，从架构层面解决安全规则碎片化的问题。

## 实战洞察

这个思路对任何运行多 Agent 系统的团队都有直接价值。当前很多人在配置 AI Agent 时只关注功能和人格，安全规则要么缺失要么不一致，这在 Agent 可以执行文件操作、网络请求的环境下是真实的风险。将安全规则抽象为一个集中式文件并让所有 Agent 继承，不仅降低了维护成本，还消除了"某个 Agent 漏配安全规则"的盲区。这个模式可以立即应用到现有的 Agent 工作流中，作为基础设施层面的安全加固。
