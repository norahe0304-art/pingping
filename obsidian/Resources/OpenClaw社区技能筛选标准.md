# OpenClaw 社区技能筛选标准

## 内容摘要

1. **筛选背景**：从 ClawHub 5705 个 Skills 筛选到 3002 个，排除率接近 48%
2. **垃圾与低质量内容**（43%）：包括批量账户创建的测试 Skills、未正式发布的开发代码、功能相同但反复提交的重复版本
3. **加密与金融交易类**（24%）：所有虚拟货币、区块链、金融交易和投资工具被整体排除——风险规避
4. **功能重复**（18%）：当多个 Skills 实现相同功能时，保留更新最活跃或功能最完整的版本
5. **社区选择主动清理**：开源生态问题不可避免，但 OpenClaw 社区选择了主动清理
6. **核心启示**：质量比数量重要，不是所有 skill 都值得用
7. **金融类工具风险高**：AI Agent 可自主执行操作时，金融类工具天然带有更高责任风险

## 我们可以做什么

1. **建立自己的 Skill 评估 checklist**：制定质量评估标准
2. **金融类工具使用前必须人工审核**：避免自动执行带来的风险
3. **定期清理长期不用的 Skills**：保持环境整洁
