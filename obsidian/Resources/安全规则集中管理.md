# X 推文洞察

## Source
- Main URL: https://x.com/ohxiyu/status/2025039535500132488?s=46&t=gC1r3zh0g-l5o215P0tIzg

## 核心要点

这条推文讲的是"拒绝 OpenClaw 裸奔——SECURITY.md 下沉设计"。

核心观点是：大多数人有多个 AI Agent，每个都有人格定义、分工、权限。但安全规则呢？散落在各个配置文件里——这个 Agent 写了"不要删文件"，那个忘了；这个 Agent 知道不能碰密钥目录，那个压根没配。更要命的是，改了一条安全规则，得挨个去改每个 Agent 的配置。

作者的安全原则是：多做 1%，安全性就可能从 0% 提高到 99%。这和"千里之堤溃于蚁穴"是一个道理——安全不是做到 100% 才有效，而是每多做一点，风险就降一分。

解决方案是 SECURITY.md 下沉设计——把安全规则集中管理，每个 Agent 引用同一个安全配置，而不是各自为政。

## 实战洞察

动作：
1. 建立 SECURITY.md 文件，集中管理安全规则
2. 所有 Agent 配置引用同一个安全配置
3. 每次改规则只需要改一处
4. 定期 review 安全规则是否有漏洞
