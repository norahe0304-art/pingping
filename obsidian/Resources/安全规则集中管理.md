# 安全规则集中管理 - SECURITY.md 下沉设计

## 内容摘要

1. **问题背景**：多个 AI Agent 各自有人格定义、分工、权限，但安全规则散落在各个配置文件里
2. **混乱现状**：这个 Agent 写了"不要删文件"，那个忘了；这个知道不能碰密钥目录，那个没配
3. **维护困难**：改了一条安全规则，得挨个去改每个 Agent 的配置
4. **安全原则**：多做 1%，安全性就可能从 0% 提高到 99%
5. **蚁穴效应**：安全不是做到 100% 才有效，每多做一点风险就降一分
6. **解决方案**：SECURITY.md 下沉设计——安全规则集中管理
7. **引用机制**：每个 Agent 引用同一个安全配置，而不是各自为政

## 我们可以做什么

1. **建立 SECURITY.md 文件**：集中管理安全规则
2. **统一引用配置**：所有 Agent 配置引用同一个安全配置
3. **单点修改**：确保每次改规则只需要改一处
4. **定期 review**：检查安全规则是否有漏洞
